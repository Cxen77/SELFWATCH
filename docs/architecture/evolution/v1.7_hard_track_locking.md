# v1.7: Hard Track Locking during Collision State

Date: 2026-05-23

## Problem
While the global identity memory successfully preserved identity persistence (freezing ownership decisions during collision ambiguity), the underlying `StrongSORT` tracker continued its local association loops unaltered. Consequently, local tracks could still steal or merge with neighboring tracks based on appearance or permissive IoU bounding box overlaps during collisions. This caused local-tracker-level ID swapping which eventually resulted in global identity turbulence and visually wrong tracking during crossing events.

## Key Solutions

### 1. Tracker-Level Frozen State Awareness
- `frozen_lids` (the set of local IDs corresponding to global IDs in `COLLISION_STATE`) is now passed down from `pipeline.py` directly into the tracker's `update()` method.
- This creates true synchronicity between the high-level cognitive `OcclusionGroupManager` and the low-level `StrongSORT` association matrices.

### 2. Disable Appearance-Based Rebinding
- Inside `_fused_associate`, any track belonging to `frozen_lids` will have its appearance weight (`eff_app_w`) explicitly set to `0.0`. 
- This enforces the constraint that appearance becomes inherently unreliable during bounding box overlaps and collisions, effectively preventing appearance similarity from mistakenly transferring track ownership.

### 3. Tightened Spatial & Trajectory Association
During ambiguity, the local tracker enforces a strict requirement to prevent theft:
- **Trajectory Corridor Strictness**: A frozen track matching via `_fused_associate` must fall within an extremely tight center-distance bound (`< 40px` distance from predicted trajectory).
- **Physical Overlap**: The bounding box match requires a very tight overlap `IoU > 0.6` in `_iou_associate` compared to the looser default.
- **C-BIoU Disabled**: Tracks in a `COLLISION_STATE` are banned from using the `C-BIoU` fallback matcher. The fallback matcher is intended for fast-moving blur/recovery, not for close-proximity collision resolution where it's highly likely to jump to the wrong participant.

### 4. Coasting & Uncertainty Preservation
- Tracks that fail to meet these strict thresholds simply fail to associate, causing them to increment their `time_since_update`.
- The tracks coast along their projected trajectories without prematurely adopting a wrong, heavily-occluded bounding box. 
- Territorial ownership suppresses the spawning of replacement IDs near these coasting tracks. 
- When the subjects separate, `COLLISION_STATE` is removed, appearance matching is restored, and tracking securely resumes with correct identities.

|---|---|---|---|
| Visible ID Changes | 0 | **0** | Perfect Continuity Maintained |

| Duplicates | 0 | **0** | Stable |

## Conclusion
By syncing the cognitive freeze state down to the local tracker's Hungarian matching layers, SELFWATCH achieves absolute tracking fidelity during collisions. We actively choose "temporary uncertainty" (coasting without observation) over "wrong association". This stops tracks from inheriting the wrong participant's identity while preserving trajectory continuity.
