# Architecture Evolution: v0.6 — Ownership Arbitration Layer

Date: 2026-05-23

## Core Problem

The SELFWATCH tracker leaked internal ownership uncertainty into visual output.

During crowd occlusion:
- Multiple ownership hypotheses were rendered simultaneously
- ACTIVE and THINKING tracks for the same spatial region created duplicate boxes
- Temporary ID oscillation occurred even when long-term identity was preserved

The system correctly preserved **final** identity ownership, but failed at
**temporary visual continuity** during ambiguity.

## Architecture Change: Single Visual Identity Dominance

### Previous Architecture (v0.5)
```
Detection → Tracking → Identity → Memory → Render ALL active hypotheses
```

### New Architecture (v0.6)
```
Detection → Tracking → Identity → Memory → ARBITRATE → Render WINNER only
                                              ↓
                                         Shadow hypotheses (internal only)
```

## New Components

### 1. Ownership Arbitration Layer (`memory/ownership_arbitration.py`)

Three integrated systems:

**a) Visual Dominance Resolver**
- Detects spatially overlapping identities (IoU > 0.20)
- Picks single dominant identity per region using priority hierarchy:
  - ACTIVE always beats THINKING
  - Frozen identity wins (was there first)
  - Higher stability counter wins
  - Older track wins
  - Deterministic tiebreak (lower gid)

**b) Shadow Hypothesis Tracker**
- Suppressed identities are NOT deleted — they become shadow hypotheses
- Shadow hypotheses persist internally for up to 30 frames
- If the dominant identity fails, a shadow can seamlessly take over
- Humans never see the shadow — only the dominant

**c) Ambiguity Scorer**
- Measures per-identity association entropy from cost matrix
- When ambiguity is high (multiple detections compete for one track), delays rebinding
- Prevents snap decisions during momentary confusion

### 2. Track-Aware NMS (in `trackers/strongsort_tracker.py`)

New birth suppression stage that checks ALL confirmed active tracks (age > 5)
before spawning a new local track. Prevents duplicate identity generation at the source.

### 3. Enhanced Frozen Ownership (from v0.5.1)

- Post-freeze cooldown (8 frames)
- Exit trajectory prediction and plausible recovery gating
- Source-level proposal suppression for frozen/cooldown GIDs
- 5-frame inertia for post-frozen rebinding

## Design Philosophy

> "The human viewer should see ONE stable identity per person, ALWAYS.
>  Internal uncertainty is fine. VISUAL uncertainty is a failure."

This is the fundamental shift from v0.5: the system now treats
**visual output** as a distinct optimization target, separate from
**internal tracking consistency**.

## Priority Hierarchy

```
ACTIVE (real detection, confirmed) > FROZEN (ownership locked)
  > ESTABLISHED (high stability counter) > OLDER (higher track age)
    > DETERMINISTIC (lower gid tiebreak)
```

THINKING tracks are always suppressed when they overlap with ACTIVE tracks.
Phantom tracks are never rendered (existing behavior preserved).

## Impact

Expected improvements:
- Dramatic reduction in duplicate box frames
- Elimination of visual ID oscillation during crowd overlap
- Stable single-owner visual identity during ambiguity
- Internal hypotheses preserved for seamless recovery
