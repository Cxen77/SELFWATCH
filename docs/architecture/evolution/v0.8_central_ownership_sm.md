# Architecture Evolution: v0.8 — Centralized Ownership State Machine

Date: 2026-05-23

## Problem Statement

The SELFWATCH system developed architectural fragmentation where 6+ subsystems
independently managed identity ownership state, creating contradictory behavior:

| Subsystem | State it managed | Scope |
|---|---|---|
| `strongsort_tracker.py` | `is_confirmed/lost/tentative` | Local tracking |
| `pipeline._id_states` | `ACTIVE/THINKING/PHANTOM/DEAD` | Identity lifecycle |
| `global_identity.py` | `_local_to_global`, proposals | Identity mapping |
| `occlusion_groups.py` | `frozen_gids`, cooldown | Frozen state |
| `ownership_arbitration.py` | `_visual_owners`, shadows | Visual dominance |
| `evaluator.py` | Independent per-frame counters | Metric measurement |

### Metric Contradiction

```
Internal metrics:  "ID switches: 0, Continuity: 100%"
Visual metrics:    "Visible switches: 22, Stability: 90.2%"
```

Both correct for their scope, but the system reported both as if measuring
the same thing. Users saw "perfect" metrics while observing visual instability.

## Solution: Central Ownership State Machine

### New Module: `memory/ownership_state.py`

Single canonical authority for identity ownership decisions.

**States:**
```
VISIBLE_ACTIVE    — Real detection, confirmed, currently rendered
VISIBLE_FROZEN    — In occlusion group, rendered but locked
LATENT_CANDIDATE  — Track lost, within hold window, NOT rendered
RECOVERING        — Recovery accepted, visual hysteresis pending
ARCHIVED          — Moved to warm memory, no longer tracked
```

**Key Properties:**
- Every identity has exactly ONE canonical state
- All state transitions are logged forensically
- Other subsystems QUERY state but never SET it
- Confidence scoring per identity (grows when ACTIVE, decays when LATENT)

### Responsibility Separation

| Subsystem | Does | Does NOT |
|---|---|---|
| Tracker | Spatial observations | Identity decisions |
| Ownership SM | All identity reasoning | Rendering, metrics |
| Renderer | Visual continuity | Identity decisions |
| Evaluator | Independent measurement | State changes |

### Metric Consistency Fix

Pipeline close now clearly labels:
```
  INTERNAL TRACKER METRICS (association-level)
  NOTE: These measure internal consistency, NOT visual output.
```

And separately:
```
  OWNERSHIP STATE MACHINE — Final State
  VISIBLE_ACTIVE: N
  LATENT_CANDIDATE: N
  (recent transition log)
```

## Integration Approach

Backward-compatible integration alongside the existing `_id_states` dict:
- Central SM is initialized in pipeline constructor
- SM transitions called alongside legacy `_set_state` calls
- SM summary exposed in return dict and periodic status print
- Legacy system will be gradually deprecated as SM proves stable

## Design Decision: Why Not Replace _id_states Immediately?

The legacy `_id_states` dict is deeply integrated into the render pipeline,
memory transitions, and thinking-identity retrieval. Replacing it all at
once risks breaking the entire pipeline. Instead:

1. v0.8: Add SM alongside legacy (current)
2. v0.9: Migrate render pipeline to use SM
3. v1.0: Remove legacy _id_states entirely
