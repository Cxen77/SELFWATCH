# Architecture Evolution: v1.1 — Cognitive Simplification

Date: 2026-05-23

## Problem Statement

The SELFWATCH architecture had accumulated too many overlapping mechanisms:
- Challenger competition systems
- Dominance arbitration scoring
- Tiered momentum stacking
- Multi-level hysteresis chains
- Confidence scoring
- Shadow hypothesis tracking
- Candidate sandbox testing

These produced worse visual results than the earlier cognitive-layer
approach despite being much more complex.

## Philosophy

Returned to the core cognitive principle:

> "This is person X until there is overwhelming evidence otherwise."

## What Was Removed

### From ownership_arbitration.py (466 → ~200 lines)
- ❌ Dominance score computation
- ❌ Confidence scoring & decay
- ❌ Challenger frame counting
- ❌ Hysteresis chains (12-frame challenger requirement)
- ❌ Confidence margin (0.25 threshold)
- ❌ Candidate sandbox (5-frame test)
- ❌ Visual lock threshold (60-frame bonus)
- ❌ Shadow hypothesis tracker

### From global_identity.py
- ❌ Tiered inertia constants (THINKING=6, PHANTOM=5, FULL=8)
- ❌ Ownership momentum dict (per-lid frame counting)
- ❌ Momentum bonus computation (0/2/4/6 frame bonuses)
- ❌ Source-specific inertia paths (thinking vs phantom vs warm)

## What Was Kept

### ownership_arbitration.py — now does exactly two things:
1. Detect spatial overlaps (IoU > 0.20)
2. Keep senior identity, suppress junior (seniority = stability + ACTIVE state)

### global_identity.py — simple committed-track protection:
1. New tracks get provisional window (5 frames)
2. During provisional: proposals CAN change ownership
3. After commit: ownership is STICKY — all proposals rejected
4. Ownership changes only through natural lifecycle:
   - Track dies → new track spawns
   - Owner archived from warm memory
   - Impossible trajectory (still checked before commit)

### Central Ownership State Machine (unchanged)
- Canonical state authority
- Forensic transition logging
- Recovery hysteresis (8 frames)

### Evaluator (unchanged)
- Independent measurement
- min_visible_frames=15

## Results Comparison

| Metric | Complex (v0.8-v1.0) | Simplified (v1.1) |
|---|---|---|
| Lines in arbitration | 466 | ~200 |
| Lines in identity (net) | ~540 | ~480 |
| Mechanisms | 7+ | 2 |
| Visible Switches | 57-121 | **64** |
| Stability | 0.76-0.87 | **0.86** |
| Internal ID Switches | 0 | **0** |
| FPS | 18.5 | **17.8** |
| Duplicate Boxes | 0 | **0** |

## Design Decision: Simplicity Over Optimization

The complex system (v0.8-v1.0) achieved slightly fewer visible switches (57)
but at the cost of 7+ interacting mechanisms that were:
- Hard to debug
- Hard to tune
- Producing contradictory behavior
- Amplifying incorrect assignments

The simplified system (v1.1) achieves comparable results with 2 mechanisms
that are trivially understandable and compose cleanly.
