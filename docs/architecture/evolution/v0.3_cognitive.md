# v0.3 — Cognitive Memory System

**Date**: 2026-05
**Git Tag**: `v0.3.0` (retroactive)

## Motivation

Standard trackers forget identities after occlusion. A persistent memory system was needed to maintain identity across disappearances and re-appearances.

## What Changed

- Active Memory: short-term identity state for visible tracks
- Warm Memory: medium-term persistence for recently lost tracks
- Identity Fingerprint: multi-signal identity encoding (embedding + gait + topology)
- Global Identity Mapper: consistent ID assignment across track lifecycle
- Cognitive Engine: central reasoning hub for identity decisions
- Event Logger: JSONL-based cognitive event logging
- Forensic Debugger: auto-capture of failure cases (clip + frame + metadata)
- 6-weight confidence fusion scoring
