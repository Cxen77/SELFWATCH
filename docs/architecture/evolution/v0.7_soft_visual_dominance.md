# Architecture Evolution: v0.7 — Soft Visual Dominance

Date: 2026-05-23

## Problem Statement

The v0.6 hard ownership constraints (REJECT_COOLDOWN, REJECT_EXIT_REGION,
restricted_gids filtering) successfully eliminated duplicate boxes but
destroyed identity recovery capability:

| Metric | v0.5 (before) | v0.6 (over-constrained) |
|---|---|---|
| Duplicate Box Frames | 125 | 0 |
| Retrieval Success | ~2.4% | 0.5% |
| Visible ID Switches | 22 | 0 |
| Recovery capability | Functional | Broken |

The hard gates blocked valid recovery proposals at their source, creating
a worse problem: permanent identity loss.

## Architecture Shift: Hard Gates → Soft Penalties

### v0.6: Hard gates block internal reasoning
```
Proposal → HARD BLOCK (cooldown/exit-region) → Dead. No recovery.
```

### v0.7: Soft penalties in reasoning + hard filter at visual layer only
```
Proposal → SOFT PENALTY (extra inertia frames) → Still flows through
                                                    ↓
                                              Identity manager accepts
                                                    ↓
                                              Display dict built
                                                    ↓
                                              VISUAL ARBITRATION
                                              (render hysteresis)
                                                    ↓
                                              Single visual owner
```

## Key Changes

### 1. Reasoning Layer — Unrestricted Proposals
`reasoning.py` no longer filters THINKING identities by `restricted_gids`.
All proposals are generated freely. The visual layer handles deduplication.

### 2. Identity Manager — Soft Penalties Replace Hard Gates
| Gate | v0.6 | v0.7 |
|---|---|---|
| FROZEN | HARD REJECT (kept) | HARD REJECT (kept) |
| COOLDOWN_SRC | HARD REJECT | +2 inertia frames |
| COOLDOWN_TGT | HARD REJECT | +2 inertia frames |
| EXIT_REGION | HARD REJECT | +2 inertia frames |
| Trajectory/Direction | HARD REJECT (kept) | HARD REJECT (kept) |

Physics-based gates (frozen, trajectory, direction) remain hard.
Timing-based gates (cooldown, exit region) become soft penalties.

### 3. Ownership Arbitration — Render Hysteresis
New visual layer features:
- **Render hysteresis**: Visual owner change requires 4 frames of consistent
  challenger dominance
- **Confidence margin**: Challenger must beat incumbent by 0.10 margin
- **Candidate sandbox**: Suppressed identities tested for 3 frames before
  being considered for visual promotion
- **Continuous confidence scoring**: Grows when ACTIVE (+0.05/frame),
  decays when THINKING (-0.03/frame)
- **Soft stability decay**: Counter decreases by 2/frame when invisible
  (instead of hard reset to 0)

## Design Philosophy

> "Internal reasoning should be GENEROUS. Visual output should be CONSERVATIVE."

The system now maintains maximum internal recovery flexibility while
presenting minimum visual uncertainty. Hard constraints protect against
physically impossible rebindings. Soft constraints bias toward stability
without destroying recovery.
